# 数据类型-MySQL
## 1. 数据值类别
### 1.1 数值
MySQL 能识别的数值包括整数/浮点数/定点数/位域值。

- 整数表示

  整数可以表示为十进制或十六进制。

  - 十进制下是一个只包含数字的序列。
  
  - 十六进制下是由 `0x` 和数字组成的序列，默认会被识别为字符串，但是进行数值计算时会被视为 64 位整形。
  
- 精确值/近似值表示
  
  精确值即由小数点和数字组成的确定的值，例子：
  ```
  10
  9.05
  ```

  MySQL 也支持使用科学计数法表示的近似值，例子：
  ```
  1.234E10
  0.233E-6
  ```
  
  注意科学计数法只能用于表示十进制数。
- 位域值

  `BIT` 可以表示一连串的二进制数，表示方法有两种，`b'value'` 或者 `0bvalue`，例子：
  ```
  b'1001'
  0b1001
  ```
  
  位域值在查询结果集中也是二进制串表示，如果要输出十进制整数表示可以使用 `CAST()` 函数或者直接在表达式后面 `+ 0`，例子：
  ```
  b'1001' + 0 # 结果是 9
  CAST(b'1001' AS UNSIGNED) # 结果是 9
  ```
  
### 1.2 字符串值
SQL 标准规定了字符串使用单引号表示字符串值，双引号也可以表示字符串值，但是为了移植，还是推荐使用单引号。

另外 MySQL 中如果启用了 `ANSI_QUOTES` 模式，会把双引号中的字符串识别为标识符（表名或者数据库名）而不是单纯的字符串。

#### 1.2.1 转义
转义序列以 `\` 反斜杠字符开始，有如下转义字符：
|转义序列|含义|
|---|---|
|`\0`|`NUL` 零值字节|
|`\'`|单引号|
|`\"`|双引号|
|`\b`|退格符|
|`\n`|换行符|
|`\r`|回车符|
|`\t`|制表符|
|`\\`|反斜杠|
|`\Z`|`Ctrl + Z` 是 Windows 系统中的 `EOF` 字符|

需要注意，转义序列中的字母是区别大小写的。

#### 1.2.2 字符集
字符串值一般分为两类：
* 二进制串，可以看做一个字节序列，与字符集无关。
* 非二进制串，与字符集强相关，字符集决定了非二进制串能使用的字符。同时字符集和排序规则决定非二进制串的排序时的顺序，默认的字符集和排序规则为 `latin1` 和 `latin1_swedish-ci`。

可以用以下命令查询当前数据库支持的字符集：
```
SHOW CHARACTER SET;
```

可以用以下命令查询对应字符集支持的排序规则：
```
SHOW COLLATION;
```

一种字符集有一种或多种排序规则，但只有一种默认排序规则。

规则后缀一般有和规则对应的含义，比如：
```
_ci 表示排序不分大小写
_cs 表示排序分大小写
_bin 表示基于字符编码值排序，与具体语言无关
```

### 1.3 时态
分为日期值和时间值：
```
'2021-08-26' # 日期
'20:20:34' # 时间
'20:20:34.005' # 时间带小数的秒，最多可以有 6 位（即微秒）
```

也可以组合一起使用：
```
'2021-08-26T20:20:34.005' # 组合表示，中间用`T` 字符分隔，不能使用空格
```

### 1.4 空间值
MySQL 的 InnoDB、MyISAM、NDB、ARCHIVE 这四种引擎支持空间值，可以直接表示点、线、多边形这样的空间数据。

这些值都遵循 OpenGIS 规范，例如给自定义变量赋值为一个 (10,20) 坐标的点：
```
SET @point = POINTFROMTEXT('POINT(10 20)');
```

### 1.5 布尔值 
* `TRUE` 在计算中被当做 `1`。表达式中任何非零或者非 `NULL` 值在判断时都作为 `TRUE`。
* `FALSE` 在计算中被当做 `0`。表达式中零值在判断时作为 `FALSE`。

布尔值表示不区分大小写。

### 1.6 NULL 值
`NULL` 表示 **没有类型** 的值，`NULL` 是可以检测的：
```
SELECT ISNULL(NULL);
```

但是不能参与算术，任何 `NULL` 参与的算术运算结果都是 `NULL`。

MySQL 中可以用 `\N` 表示 `NULL`。

## 2. 数据类型
### 2.1 数值类型
- 整数
  |关键字|存储空间大小（字节）|
  |---|---|
  |TINYINT|1|
  |SMALLINT|2|
  |MEDIUNINT|3|
  |INT|4|
  |BIGINT|8|
  
  这里存储空间大小是固定的，比如在定义属性 `INT(12)` 时，该属性存储空间仍然是 32，不过结果集和交互工具显示数字的长度为 12。
  
- 浮点数/定点数
  
  |关键字|描述|存储空间大小（字节）|
  |---|---|---|
  |FLOAT|单精度浮点数|4|
  |DOUBLE|双精度浮点数|8|
  |DECIMAL|定点数|根据 DECIMAL 声明时的长度决定，整数/小数部分每9位数字需要32位，其他剩余部分需要8~32位，需要空间为整数和小数两部分总和|
  
  浮点/定点数声明的格式都是整数+小数部分长度，例如：
  ```
  # 都表示 10 位整数，3 位小数
  FLOAT(10,3)
  DOUBLE(10,3)
  DECIMAL(10,3)
  ```
  
  DECIMAL 值不是 CPU 原生支持的，计算代价较大。
  
- 位域值

  `BIT` 长度也取决于声明的长度(m+7)/8，例如：
  ```
  BIT(8) # 需要 2 字节
  ```
  
### 2.2 字符串类型
以`w` 表示字符集中最宽的字符所占的字节数，以 `L` 表示某个给定值(即当前行的值)的字节长度，下表中使用 `L` 表示当前存储空间大小时意味着当前类型是可变长的类型。

基本的字符串类型：
|关键字|最大长度|存储空间大小（字节数）|
|---|---|---|
|CHAR[(M)]|M 个字符|M * w|
|VARCHAR(M)|M 个字符|L + 1 或 L + 2|
|TEXT|2^16 - 1 个字节|L + 2|
|TINYTEXT|2^8 - 1 个字节|L + 1|
|MEDIUMTEXT|2^24 - 1 个字节|L + 3|
|LONGTEXT|2^32 - 1 个字节|L + 4|

二进制串类型：
|关键字|最大长度|存储空间大小（字节数）|
|---|---|---|
|BINARY[(M)]|M 个字符|M|
|VARBINARY(M)|M 个字符|L + 1 或 L + 2|
|BLOB|2^16 - 1 个字节|L + 2|
|TINYBLOB|2^8 - 1 个字节|L + 1|
|MEDIUMBLOB|2^24 - 1 个字节|L + 3|
|LONGBLOB|2^32 - 1 个字节|L + 4|

### 2.3 时态类型
|关键字|取值范围|零值|存储空间大小（字节）|
|---|---|---|---|
|DATE|`1000-01-01` ~ `9999-12-31`|`0000-00-00`|3|
|TIME|`-838:59:59[.000000]` ~ `838:59:59[.000000]`|`00:00:00[.000000]`|3 + `fsp`|
|DATETIME|`1000-01-01 00:00:00[.000000]` ~ `9999-12-31 23:59:59[.999999]`|`0000-00-00 00:00:00[.000000]`|5 + `fsp`|
|TIMESTAMP|`1970-01-01 00:00:01[.000000]` ~ `2038-01-19 03:14:07[.999999]`|`0000-00-00 00:00:00[.000000]`|4 + `fsp`|
|YEAR|`1901` ~ `2155`|0000|1|

上表的 `fsp` 表示小数部分长度
|精度|存储空间大小（字节）|
|---|---|
|0|0|
|1,2|1|
|3,4|2|
|5,6|3|
